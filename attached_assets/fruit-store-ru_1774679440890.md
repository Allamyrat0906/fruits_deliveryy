# 🍎 Fruit Store — Полная Структура Проекта

## Технологический Стек
| Слой | Технология |
|---|---|
| Frontend | React 18 + Vite |
| Стили | Tailwind CSS v3 |
| State | React Context API + useReducer |
| Маршрутизация | React Router v6 |
| Форма | React Hook Form + Zod (валидация) |
| HTTP Клиент | Axios |
| Иконки | Lucide React |
| Backend | Node.js + Express.js |
| База данных | PostgreSQL |
| ORM | Prisma |
| Авторизация | JWT (Access + Refresh Token) + **Google OAuth 2.0** |
| Пароль | bcrypt |
| Env | dotenv |
| OAuth | Passport.js (passport-google-oauth20) |

## 🗄️ PostgreSQL — Схема Базы Данных (Prisma)

```prisma
// prisma/schema.prisma

model User {
  id              Int      @id @default(autoincrement())
  name            String
  email           String   @unique
  password        String?  // ⚠️ Может быть null для Google-пользователей
  googleId        String?  @unique  // 🆕 ID из Google OAuth
  avatar          String?           // 🆕 Фото профиля из Google
  role            Role     @default(CUSTOMER)
  createdAt       DateTime @default(now())
  orders          Order[]
}

model Fruit {
  id           Int         @id @default(autoincrement())
  name         String
  slug         String      @unique
  description  String?
  price        Float                       // ₽ / кг
  stock        Int         @default(0)
  category     Category
  organic      Boolean     @default(false)
  imageUrl     String?
  createdAt    DateTime    @default(now())
  orderItems   OrderItem[]
}

model Order {
  id           Int         @id @default(autoincrement())
  user         User        @relation(fields: [userId], references: [id])
  userId       Int
  status       OrderStatus @default(PENDING)
  totalPrice   Float
  address      String
  createdAt    DateTime    @default(now())
  items        OrderItem[]
}

model OrderItem {
  id           Int    @id @default(autoincrement())
  order        Order  @relation(fields: [orderId], references: [id])
  orderId      Int
  fruit        Fruit  @relation(fields: [fruitId], references: [id])
  fruitId      Int
  quantity     Int
  weight       String   // "500г" | "1кг"
  unitPrice    Float
}

enum Role {
  CUSTOMER
  ADMIN
}

enum Category {
  МЕСТНЫЕ
  ТРОПИЧЕСКИЕ
  ОРГАНИЧЕСКИЕ
  ИМПОРТНЫЕ
}

enum OrderStatus {
  ОЖИДАНИЕ
  ПОДТВЕРЖДЁН
  ОТПРАВЛЕН
  ДОСТАВЛЕН
  ОТМЕНЁН
}
```

## 🌐 REST API Эндпоинты

### Авторизация
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/api/auth/register` | Регистрация нового пользователя |
| POST | `/api/auth/login` | Вход + возврат JWT |
| POST | `/api/auth/logout` | Инвалидация токена |
| GET | `/api/auth/me` | Данные текущего пользователя |
| GET | `/api/auth/google` | 🆕 Перенаправление на Google OAuth |
| GET | `/api/auth/google/callback` | 🆕 Обратный вызов от Google → генерация JWT |

### Фрукты
| Метод | Эндпоинт | Описание |
|---|---|---|
| GET | `/api/fruits` | Все фрукты (фильтр + пагинация) |
| GET | `/api/fruits/:slug` | Детали конкретного фрукта |
| POST | `/api/fruits` | Добавить фрукт (Администратор) |
| PUT | `/api/fruits/:id` | Обновить фрукт (Администратор) |
| DELETE | `/api/fruits/:id` | Удалить фрукт (Администратор) |

### Заказы
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/api/orders` | Создать заказ (требует авторизации) |
| GET | `/api/orders/my` | Заказы пользователя (требует авторизации) |
| GET | `/api/orders/:id` | Детали заказа (требует авторизации) |
| PATCH | `/api/orders/:id/status` | Обновить статус (Администратор) |

## 🔐 Схема Авторизации

### Обычный вход (Email + Пароль)
```
Пользователь входит
   → Сервер генерирует JWT (Access Token: 15мин, Refresh Token: 7 дней)
   → Access Token сохраняется в localStorage
   → Каждый запрос отправляет заголовок: Authorization: Bearer <token>
   → При истечении срока вызывается refresh эндпоинт
```

### 🆕 Вход через Google OAuth 2.0
```
Пользователь нажимает "Войти через Google"
   → Перенаправление на /api/auth/google
   → Google проверяет пользователя и перенаправляет на /api/auth/google/callback
   → Сервер ищет пользователя по googleId в БД
        → Если существует: авторизуется
        → Если нет: создаётся новый User с googleId + avatar
   → Генерируется JWT и возвращается клиенту
   → Клиент сохраняет токен, AuthContext обновляется

⚠️ Важно: для Google OAuth в Google Cloud Console необходимо
   создать OAuth 2.0 Credentials и добавить
   http://localhost:5000/api/auth/google/callback
   в список разрешённых Redirect URIs.
```

## ▶️ Запуск

```bash
# Терминал 1 — Backend
cd server
npm install
npm install passport passport-google-oauth20
npx prisma migrate dev
npx prisma db seed
npm run dev          # localhost:5000

# Терминал 2 — Frontend
cd client
npm install
npm run dev          # localhost:5173
```
